# 开发者模式使用说明

## 功能说明

开发者模式是一个系统级别的开关，开启后可以绕过部分业务校验，用于处理脏数据和特殊情况。

**⚠️ 重要提示**：开发者模式仅用于开发环境和数据处理，生产环境请勿开启！

## 使用场景

1. **删除已完成/已结算的订单**：正常情况下已完成或已结算的订单不能删除，开启开发者模式后可以强制删除
2. **处理脏数据**：用于清理测试数据或异常数据
3. **调试测试**：开发测试时绕过业务限制

## 开启方式

### 步骤一：执行数据库脚本

```bash
mysql -u your_user -p your_database < sql/migrations/v3.2.1_add_developer_mode.sql
```

### 步骤二：在系统管理中开启

1. 登录系统
2. 进入 **系统管理 > 参数设置**
3. 找到参数 **`sys.developer.mode`**（开发者模式）
4. 将参数值修改为 `true`
5. 点击保存

## 关闭方式

在参数设置中将 `sys.developer.mode` 改回 `false` 即可。

## 使用示例

### 删除已完成的订单

**开启开发者模式后**：

1. 进入 **物流管理 > 订单管理**
2. 找到需要删除的订单（包括已完成状态的订单）
3. 点击删除按钮
4. 确认删除

**订单日志中的记录**：
- 正常删除：操作内容为"删除订单"
- 开发者模式：操作内容为"开发者模式强制删除订单"

## 技术实现

### 受影响的模块

| 模块 | 功能 | 开发者模式效果 |
|------|------|----------------|
| 订单管理 | 删除订单 | 绕过状态校验（completed、settled） |
| 回单管理 | 删除回单 | 恢复订单状态为 pending |

### 代码实现

```java
// 检查是否开启开发者模式
String developerMode = configService.selectConfigByKey("sys.developer.mode");
boolean isDeveloperMode = "true".equals(developerMode);

// 非开发者模式下才进行业务校验
if (!isDeveloperMode) {
    if ("completed".equals(order.getOrderStatus())) {
        throw new ServiceException("已完成的订单不能删除");
    }
}
```

## 注意事项

1. **安全性**：开发者模式会绕过重要的业务校验，请谨慎使用
2. **数据完整性**：强制删除可能导致数据关联异常，建议先检查关联数据
3. **操作日志**：所有删除操作都会记录在订单操作日志中，便于追溯
4. **生产环境**：生产环境务必关闭此功能

## 故障排查

### 开启后仍无法删除

1. 检查参数是否正确保存（值必须为 `true`）
2. 清除 Redis 缓存或重启应用
3. 检查是否有其他权限限制

### 参数不存在

执行 SQL 脚本：
```sql
INSERT INTO `sys_config` (`config_name`, `config_key`, `config_value`, `config_type`, `remark`)
VALUES ('开发者模式', 'sys.developer.mode', 'false', 'Y', '开发者模式：开启后可绕过业务校验强制删除数据');
```
