## 二、基础数据管理流程

### 2.1 客户管理流程

```mermaid
flowchart LR
    A[新增客户] --> B[填写客户信息]
    B --> C[客户编码唯一性校验]
    C -->|重复| D[提示编码重复]
    C -->|通过| E[保存客户信息]
    E --> F[设置信用额度]
    F --> G[选择结算方式]
    G --> H[客户创建成功]
```

**关键业务规则**：
- 客户编码必须唯一
- 结算方式：月结/现结
- 信用额度用于控制客户欠款上限

### 2.2 司机/车队管理流程

```mermaid
flowchart TD
    A[司机管理入口] --> B{操作类型}
    B -->|新增车队| C[填写车队信息]
    B -->|新增司机| D{司机类型}
    B -->|编辑/删除| E[操作已有记录]

    C --> F[车队老板信息]
    F --> G[车队结算账户]
    G --> H[保存车队]

    D -->|个人司机| I[填写个人司机信息]
    D -->|车队司机| J[选择所属车队]
    I --> K[个人银行账户]
    J --> L[无个人账户]

    K --> M[保存司机]
    L --> M

    E --> N{删除操作}
    N -->|删除车队| O{检查关联司机}
    N -->|删除司机| P[直接删除]
    O -->|有司机| Q[提示: 有关联司机]
    O -->|无司机| R[删除成功]
```

**关键业务规则**：
- 个人司机：有个人银行账户，运费结算到个人
- 车队司机：关联车队，无个人银行账户，运费结算到车队
- 删除车队前必须检查是否有关联司机
- 树形表格展示：车队（可展开）> 车队司机 + 个人司机

### 2.3 车辆管理流程

```mermaid
flowchart LR
    A[新增车辆] --> B[车牌号唯一性校验]
    B -->|重复| C[提示车牌号重复]
    B -->|通过| D[填写车辆信息]
    D --> E[设置载重/车长]
    E --> F[关联默认司机]
    F --> G[保存车辆]
```

## 三、订单管理流程

### 3.1 订单生命周期

```mermaid
stateDiagram-v2
    [*] --> 待运输: 创建订单
    待运输 --> 运输中: 开始运输
    运输中 --> 已完成: 确认回单
    已完成 --> 已开票: 开具发票
    已完成 --> 已结算: 生成结算单
    待运输 --> 已取消: 取消订单
    运输中 --> 已取消: 取消订单

    note right of 待运输
        初始状态
        可派车、可取消
    end note

    note right of 运输中
        运输进行中
        可取消
    end note

    note right of 已完成
        回单已确认
        可开票、可结算
    end note
```

### 3.2 订单创建流程

```mermaid
flowchart TD
    A[开始创建订单] --> B[选择客户]
    B --> C[填写订单信息]
    C --> D[装货地址/卸货地址]
    D --> E[添加货物明细]
    E --> F{多货物?}
    F -->|是| G[添加多个货物]
    F -->|否| H[添加单个货物]
    G --> I[自动计算总金额]
    H --> I
    I --> J[选择司机]
    J --> K{司机类型}
    K -->|个人司机| L[显示个人账户]
    K -->|车队司机| M[显示车队信息]
    L --> N[选择车辆]
    M --> N
    N --> O[设置派车日期]
    O --> P[生成订单号]
    P --> Q[保存订单]
    Q --> R{订单号生成}
    R -->|成功| S[订单创建成功]
    R -->|重复| T[重新生成序号]
    T --> P
```

**订单号生成规则**：
- 格式：`{客户编码}-{yyyyMMdd}-{序号}`
- 示例：`RMWJ-20260416-001`
- 智能防重：自动检查数据库中是否存在（包括逻辑删除记录）

### 3.3 订单状态变更流程

```mermaid
flowchart TD
    A[订单状态变更] --> B{目标状态}
    B -->|待运输→运输中| C[开始运输]
    B -->|运输中→已完成| D[确认回单]
    B -->|任何状态→已取消| E[取消订单]

    C --> F[记录操作日志]
    F --> G[更新订单状态]

    D --> H{回单状态}
    H -->|已收到| I[更新订单为已完成]
    H -->|未收到| J[提示: 请先确认回单]

    E --> K{检查条件}
    K -->|已结算| L[提示: 已结算订单不能取消]
    K -->|已开票| M[提示: 已开票订单不能取消]
    K -->|可取消| N[更新状态为已取消]
```

## 四、回单管理流程

### 4.1 回单状态流转

```mermaid
stateDiagram-v2

    未收到 --> 已收到: 确认回单
    已收到 --> [*]

    note right of 未收到
        回单已创建

    end note

    note right of 已收到
        回单已确认
        订单自动完成
    end note
```

### 4.2 回单业务流程

```mermaid
flowchart TD
    A[运输完成] --> B[创建回单]
    B --> C[生成回单号]
    C --> D[上传回单图片]
    D --> E{存储方式}
    E -->|腾讯云COS| F[上传到COS]
    E -->|本地存储| G[保存到本地]
    F --> H[获取图片URL]
    G --> H
    H --> I[保存回单记录]
    I --> J{是否确认}
    J -->|否| K[回单状态: 未收到]
    J -->|是| L[确认回单]
    L --> M[更新回单状态: 已收到]
    M --> N[联动订单状态: 已完成]
    N --> O[记录操作日志]

    K --> P{后续操作}
    O --> P
    P -->|删除回单| Q[删除回单记录]
    Q --> R[订单状态: 运输中]
    R --> S[记录操作日志]
```

**回单号生成规则**：
- 格式：`HD + 年月日 + 4位序号`
- 示例：`HD202604140001`

**回单状态联动规则**：
- 创建回单：订单状态不变
- 确认回单：订单状态自动变为"已完成"
- 删除回单：订单状态恢复为"运输中"

## 五、发票管理流程

### 5.1 发票批次状态流转

```mermaid
stateDiagram-v2
    [*] --> 草稿: 创建发票批次
    草稿 --> 已开具: 开具发票
    草稿 --> [*]: 取消合并/删除
    已开具 --> 已作废: 作废发票
    已开具 --> [*]: 不能删除
    已作废 --> [*]: 不能删除

    note right of 草稿
        可删除、可取消合并
        删除后订单恢复未开票
    end note

    note right of 已开具
        不能删除
        只能作废
    end note
```

### 5.2 合并开票流程

```mermaid
flowchart TD
    A[开始] --> B{操作入口}
    B -->|订单管理| C[勾选多个订单]
    B -->|发票管理| D[点击新增]

    C --> E{验证订单}
    E -->|存在未完成订单| F[提示: 只有已完成的订单才能开票]
    E -->|存在已开票订单| G[提示: 订单已开票]
    E -->|客户不一致| H[提示: 订单必须属于同一客户]
    E -->|验证通过| I[打开合并开票弹窗]

    D --> J[选择客户]
    J --> K[筛选该客户的未开票订单]
    K --> L[勾选多个订单]
    L --> I

    I --> M[填写开票信息]
    M --> N[开票日期]
    M --> O[发票类型<br/>普通发票/增值税发票]
    M --> P[税率%]

    N --> Q[自动计算]
    O --> Q
    P --> Q
    L --> Q

    Q --> R[开票金额 = 订单金额之和]
    Q --> S[税额 = 开票金额 × 税率]

    R --> T{确认提交}
    S --> T

    T -->|取消| U[关闭弹窗]
    T -->|确定| V[创建发票批次]
    V --> W[生成批次号<br/>FP+年月日+4位序号]
    W --> X[创建发票明细记录]
    X --> Y[更新订单状态为已开票]
    Y --> Z[记录操作日志]
    Z --> AA[提示: 合并开票成功]

    F --> BB[结束]
    G --> BB
    H --> BB
    U --> BB
    AA --> BB
```

**发票批次号生成规则**：
- 格式：`FP + 年月日 + 4位序号`
- 示例：`FP202604210001`

**订单开票状态流转**：
```
未开票 → 已开票
已开票 → 未开票 (删除发票/取消合并)
```

### 5.3 发票操作权限

| 发票状态 | 可删除 | 可取消合并 | 可作废 |
|---------|--------|-----------|--------|
| 草稿 | ✅ | ✅ | ❌ |
| 已开具 | ❌ | ❌ | ✅ |
| 已作废 | ❌ | ❌ | ❌ |

| 创建订单 | create | 新增订单时 | 订单基本信息 |

| 结算操作 | settlement | 订单结算时 | 结算信息 |

| 回单操作 | 订单状态变化 | 日志记录 |

## 八、核心业务规则总结

### 8.1 订单业务规则

1. **订单号生成**：客户编码 + 日期 + 序号（3位），智能防重
2. **多货物支持**：一个订单可包含多种货物明细
3. **状态流转**：待运输 → 运输中 → 已完成 → 已开票/已结算
4. **删除保护**：已完成、已结算的订单不能删除

### 8.2 回单业务规则

1. **回单号生成**：HD + 年月日 + 4位序号
2. **状态联动**：确认回单自动完成订单
3. **存储支持**：腾讯云COS / 本地存储
4. **多图支持**：最多上传5张回单图片

### 8.3 发票业务规则

1. **批次号生成**：FP + 年月日 + 4位序号
2. **合并开票**：支持多个订单合并开票（同客户）
3. **开票前提**：订单必须已完成且未开票
4. **操作权限**：草稿可删除，已开具只能作废

### 8.4 结算业务规则

1. **应收结算**：按客户结算已开票订单
2. **应付结算**：按司机结算已完成的运输订单
3. **司机类型**：个人司机结算到个人账户，车队司机结算到车队账户
4. **部分结算**：支持部分收款/付款

## 九、业务异常处理

### 9.1 订单异常

| 异常情况 | 处理方式 |
|---------|---------|
| 订单号重复 | 自动重新生成序号 |
| 订单已结算 | 不能取消订单 |
| 订单已开票 | 不能取消订单 |

### 9.2 回单异常

| 异常情况 | 处理方式 |
|---------|---------|
| 回单已确认 | 不能修改回单信息 |
| 删除已确认回单 | 订单状态恢复为运输中 |

### 9.3 发票异常

| 异常情况 | 处理方式 |
|---------|---------|
| 订单未完成 | 不能开票 |
| 订单已开票 | 不能重复开票 |
| 客户不一致 | 不能合并开票 |
| 发票已开具 | 不能删除，只能作废 |

## 十、数据流转关系

### 10.1 订单状态机

```
待运输 (pending)
    ├─ 开始运输 → 运输中 (transporting)
    ├─ 取消 → 已取消 (cancelled)
    └─ 派车 → 已派车 (dispatched)

运输中 (transporting)
    ├─ 确认回单 → 已完成 (completed)
    └─ 取消 → 已取消 (cancelled)

已完成 (completed)
    ├─ 开具发票 → 已开票 (invoiced)
    ├─ 生成结算 → 已结算 (settled)
    └─ 取消 → 已取消 (cancelled)

已取消 (cancelled)
    └─ 终止状态
```

### 10.2 状态字段关系

| 字段 | 取值 | 说明 |
|-----|------|------|
| order_status | pending/transporting/completed/cancelled | 订单状态 |
| receipt_status | not_received/received | 回单状态 |
| invoice_status | not_invoiced/invoiced | 开票状态 |
| settlement_status | unsettled/partial/settled | 结算状态 |
| dispatch_status | not_dispatched/partial_dispatched/dispatched | 派车状态 |
