## 二、司机/车队管理（v3.1重构）✅ 已完成

### 2.1 数据库变更

**新增车队表**：`logistics_fleet`
- 车队ID、车队名称、车队老板姓名、老板联系电话
- 车队开票账户名称、车队开票账号、车队开户行
- 状态、备注等

### 2.4 核心功能

1. **树形表格展示**
   - 车队（可展开）显示所属车队司机
   - 个人司机直接在根层级显示
   - 图标区分：🏢 车队、👤 个人司机

2. **双新增按钮**
   - 新增车队：打开车队信息表单
   - 新增司机：打开司机信息表单

3. **智能编辑功能**
   - 车队行：点击编辑车队信息
   - 司机行：点击编辑司机信息

4. **搜索功能**
   - 支持搜索车队名称
   - 支持搜索司机姓名

5. **删除保护**
   - 删除车队前检查是否有关联司机
   - 有司机关联的车队无法删除

### 2.5 业务规则

**个人司机**：
- 有个人银行账户（银行账号、开户行、账户姓名）
- 运费直接结算到个人账户

**车队司机**：
- 关联到车队
- 无个人银行账户
- 运费结算到车队账户

**车队**：
- 可以有多个司机
- 有独立的结算账户信息
- 删除前需要检查是否有司机关联

### 3.5 记录的操作类型

| 操作类型 | 代码 | 触发时机 |
|----------|------|----------|
| 创建订单 | create | 新增订单时 |
| 修改订单 | update | 修改订单任意字段时 |
| 字段变更 | update | 订单字段值发生变化时（详细记录） |
| 状态变更 | status_change | 订单状态改变时 |
| 创建回单 | update | 创建回单记录 |
| 确认回单 | status_change + update | 回单状态变为已收到 |
| 删除回单 | update + status_change | 删除回单时恢复订单状态 |
| 开票 | invoice | 订单开票时（待实现） |
| 结算 | settlement | 订单结算时（待实现） |
| 删除订单 | delete | 删除订单时 |

### 3.6 字段变更追踪

记录的字段变更：
- 订单日期
- 客户
- 装货地址 / 卸货地址
- 司机 / 车牌号 / 司机电话
- 配载单价 / 运费支出 / 代垫付金额
- 结算状态 / 付款方式 / 收款人
- 备注

### 3.7 回单状态联动

| 回单操作 | 订单状态变化 | 日志记录 |
|----------|-------------|----------|
| 创建回单 | 无变化 | 记录回单创建 |
| 确认回单 | → 已完成 | 记录状态变更 + 回单确认 |
| 删除回单 | 已完成 → 运输中 | 记录状态恢复 + 回单删除 |

### 3.8 BigDecimal 比较优化

使用 `compareTo()` 方法代替 `equals()` 进行数值比较，避免 `0.00` 和 `0` 被判定为不同值。
